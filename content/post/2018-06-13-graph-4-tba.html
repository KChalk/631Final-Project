Plot Type: Radar

How to read it and what to look for

The radar chart has multiple axes in a radial format. It allows visualization of high dimensional data through the use of polar coordinates. The chart includes n evenly spaced lines radiating out from the center of a circle where n is the number of dimensions to be shown on the chart. This chart is less effective for communicating exact values than something like the bar chart, but seems to be one of the best options for representing points in high dimensional space.

Presentation Tips

This is not a common graph, and it’s important to not interpret the area of the chart as its major component. Explaining each of the axes and what they mean is important, and this graph should only be used for high level comparisions between overall shapes of the data, rather than fine grained differences in values.

Variations and Alternatives

This graph doesn’t convey numerical values information in a very straightforward way, so it’s used here as a sort of a last resort for high dimensional data. For a more reasonable number of features a grouped bar chart might be able to better communicate differences in points across features.

Plot Description

Representaton

The create these graphs, I’ve first clasified subreddits into 7 groups using Kmeans clustering, and then plotted the resulting cluster centers the PCA space with one principle component on each axis of the radar chart. Because the principal components can be challenging to interpret, I’ve labeled each of the dimensions with an approximation of what dictionaries they correlate to.

The actual values used to plot points along these dimensions are largely arbitrary. Not only are the clusters generated based on the normalized frequency values, the values within these clusters have been normalized once again so that the smallest point each dimension is at the center of the radar plot, and the largest is on the outer circle to best express the difference between clusters.

clusters <- clusters %>%  
  mutate_at(vars(starts_with('d')),rescale)

As such I’ve removed all the numerical labels from these points and only left the line at 50% radius for reference.
